# ADR-001: WooCommerce как единый источник правды

**Дата:** 2026-03 (ретроспективно)
**Статус:** accepted
**Автор:** #5 Principal Backend (ретроспективно)

## Контекст
EDMI уже имеет работающий интернет-магазин на WordPress + WooCommerce (edmi.com.ua). Мобильное приложение должно работать с теми же товарами, заказами и клиентами. Вопрос: где хранить данные — дублировать в своей БД или использовать WC как источник правды?

## Варианты

**A: Полная репликация в свою БД**
- Плюсы: независимость, быстрые запросы
- Минусы: сложная двусторонняя синхронизация, конфликты данных

**B: WooCommerce = source of truth, middleware = кеш + прокси**
- Плюсы: единый источник, нет конфликтов, сайт и приложение синхронизированы
- Минусы: зависимость от WC API availability

## Решение
Выбрано **B**. WooCommerce — единый источник правды. Middleware кеширует в Redis (5 мин TTL) и проксирует. Webhook-first синхронизация с fallback polling каждые 5 мин.

## Последствия
- Все цены, остатки, описания управляются в WC admin
- Middleware не пишет в WC без явного действия пользователя (заказ)
- При недоступности WC — приложение показывает кешированные данные
- Server-only signatures для платежей (ключи никогда не на клиенте)
