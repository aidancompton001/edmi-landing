## Phase 0: Фундамент

**Що**: Монорепо, TypeScript типи, дизайн-токени, мок-дані, тест-інфраструктура, переклади (uk/en).

**Видимість**: Нічого в телефоні. Це фундамент на якому будується все інше.

**Що готово після Phase 0**:
- pnpm монорепо працює (`pnpm install` + `pnpm typecheck` проходять)
- Shared типи для всіх пакетів (Product, Order, Auth, etc.)
- Дизайн-токени (#b8309e, Unbounded, Inter)
- Мок-дані 15-20 товарів (структура = WooCommerce API)
- Переклади uk/en (common, products, checkout, bot, configurator)
- Jest + Vitest налаштовані
- Скелети для server, mobile, admin

**Деплой**: Нічого

## Phase 1: Mobile UI + Навігація + i18n

**Що**: Expo SDK 54+ проект, 5 табів, UI-компоненти, шрифти, кольори, i18n.

### Що видно в телефоні (Phase 1):
- 5 табів: Каталог | Конфігуратор | Наявність | Кошик | Профіль
- Правильні кольори: фіолетовий (#b8309e), синій (#0057b8), градієнт
- Правильні шрифти: Unbounded (заголовки), Inter (текст)
- UI компоненти: кнопки, картки, inputs, badges
- Переключення мови: uk / en
- Пустые экраны з placeholder-ами ("Тут буде каталог")
- FAB кнопка EDMik (placeholder)

### Як запустити:
1. `pnpm --filter mobile dev` → запускає Expo dev server
2. Встановити **Expo Go** на телефон (App Store / Google Play)
3. Сканувати QR-код з терміналу
4. Додаток відкривається на телефоні!

**Деплой**: Нічого (тільки локально через Expo Go)

## Phase 1.5: Мінімальний сервер

**Що**: Express сервер з одним endpoint GET /products, який тягне реальні товари з WooCommerce та кешує в Redis.

**Видимість в телефоні**: Нічого нового візуально, але дані починають текти з реального сайту.

**Що готово після Phase 1.5**:
- `GET /api/products` — реальні товари з edmi.com.ua
- `GET /api/products/:id` — деталі товару
- `GET /api/products/categories` — категорії
- Redis кеш (TTL 5 хвилин)
- Docker: PostgreSQL 16 + Redis 7

**Деплой сервера**: Можна задеплоїти на Railway/Render. Мінімальний сервер, один endpoint. Це дозволить мобільному додатку працювати з реальними даними навіть не на localhost.

## Phase 2: Каталог та товари

**Що**: Zustand stores, TanStack Query hooks, каталог з реальними товарами, сторінка товару, наявність.

### Що видно в телефоні (Phase 2) — ПЕРШИЙ "ВАУ" МОМЕНТ:
- Каталог з фотографіями реальних мікроскопів з edmi.com.ua
- Категорії (горизонтальний scroll)
- Фільтри та пошук
- Картки товарів: фото, бренд, назва, ціна в EUR
- Сторінка товару: карусель фото, опис, характеристики
- Кнопка "В кошик" — додає товар
- Кнопка "В обране" — серце
- Вкладка "Наявність" — товари по статусу

### Як перевірити:
- Expo Go → реальні товари з WooCommerce
- Кнопки "В кошик" та "В обране" працюють
- Фільтри та пошук працюють

**Деплой**: Сервер на Railway → мобільний показує реальні дані

## Phase 3: Корзина + Checkout + Профіль

**Що**: Повний flow покупки від корзини до підтвердження замовлення.

### Що видно в телефоні (Phase 3) — ПОВНЕ ДЕМО:
- **Корзина**: список товарів, +/-, swipe для видалення, підсумок
- **Checkout** (4 кроки):
  1. Контакти: ім'я, телефон, email
  2. Доставка: Нова Пошта / Самовивіз (мок автокомпліт)
  3. Оплата: LiqPay / WayForPay (заглушка UI)
  4. Підтвердження: номер замовлення, статус
- **Профіль**: логін/реєстрація, історія замовлень, обране, мова
- **Історія замовлень**: timeline статусів, "Замовити знову"

Після Phase 3 додаток вже можна демонструвати як повноцінний прототип.

**Деплой**: Expo Go + сервер на Railway

---

## Phase 4: EDMik — текстовий AI-бот

**Що**: FAB кнопка EDMik на всіх вкладках, текстовий чат з мок-відповідями.

### Що видно в телефоні (Phase 4):
- Фіолетова FAB кнопка (56px) на всіх екранах
- Натиснення → bottom-sheet чат
- "Привіт! Я EDMik. Чим можу допомогти?"
- Quick actions: "Повторити замовлення", "Статус замовлення", "Зв'язок з менеджером"
- Мок-відповіді на ключові слова
- "Повтори замовлення" → показує останнє → додає в кошик
- Placeholder мікрофону (неактивний, tooltip "Скоро")

**Деплой**: Expo Go

## Phase 5: Конфігуратор мікроскопа (image-based)

**Що**: Вкладка "Конфігуратор" — збірка мікроскопа з опцій.

### Що видно в телефоні (Phase 5):
- Верх: превью мікроскопа (фото, свайп між кутами, pinch zoom)
- Низ: панель конфігурації:
  - Колір (3 варіанти)
  - Кріплення (radio + ціна)
  - Освітлення (radio + ціна)
  - Об'єктив (radio + ціна)
  - Камера (toggle + ціна)
- Підсумкова ціна оновлюється в реальному часі
- "Запитати EDMik про ціну" → відкриває бот
- "Переглянути в кабінеті" → placeholder для AR

**Деплой**: Expo Go

---

## Phase 6: Повний сервер + реальні інтеграції

**Що**: Prisma schema, JWT auth, реальні платежі, Нова Пошта, Claude AI для EDMik, webhooks, push.

### Що змінюється в телефоні (Phase 6):
- EDMik відповідає через Claude API (розумні відповіді)
- Оплата через LiqPay WebView (sandbox → реальна)
- Нова Пошта: реальний автокомпліт міст та відділень
- Push-сповіщення про статус замовлення
- Авторизація через реальний WooCommerce

**Деплой**: Повний сервер на Railway/Render з PostgreSQL та Redis

---

## Phase 7: Голосовий EDMik + сповіщення

**Що**: Голосовий ввід через Whisper API, TTS відповіді, scheduled push notifications, предиктивний повтор розхідників.

### Що змінюється в телефоні (Phase 7):
- Кнопка мікрофону в EDMik чаті ПРАЦЮЄ
- Говориш → транскрипція → бот відповідає
- Бот може відповідати голосом (TTS)
- Push-сповіщення: замовлення, промо, нагадування
- "Нагадати через 14 днів замовити розхідники?"

⚠️ Потрібен **EAS Development Build** (не Expo Go!) — expo-audio-stream потребує native модулі.

**Деплой**: EAS Dev Build на телефон (через QR або direct install)

---

## Phase 8: Адмін-панель

**Що**: Веб-панель для менеджера — dashboard, AI асистент для Б/В мікроскопів, управління товарами та замовленнями.

### Що видно (веб, не телефон):
- Dashboard: статистика, графік, останні замовлення
- AI Assistant: опис Б/В мікроскопа → AI витягує модель, бренд, ціну
- Products: таблиця, фільтри, редагування наявності
- Orders: статуси, ТТН

**Деплой**: Admin на Vercel

---

## Phase 9: 3D + AR (коли моделі готові)

**Що**: 3D viewer (react-native-filament) замість фото, AR камера (ViroReact).

### Що змінюється в телефоні (Phase 9):
- Конфігуратор: 3D модель мікроскопа замість фото
- Rotate, zoom жестами
- Зміна частин в 3D (кріплення, об'єктив, камера)
- AR: "Переглянути в кабінеті" → камера → мікроскоп у реальному просторі

⚠️ Потрібен `npx expo prebuild` — ViroReact не працює в Expo Go.

**Деплой**: EAS Dev Build
