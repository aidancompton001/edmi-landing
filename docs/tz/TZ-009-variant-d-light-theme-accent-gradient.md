## Цель

Вернуть Variant D к светлой теме (белый фон, белые карточки). Фиолетовый градиент из брендбука (#90267C → #7938A9 → #11387F) использовать **только в акцентах**: цены, кнопки, бейджи, окантовки карточек при hover, hero-карточка. НЕ заливать всё чёрным.

## Указания CEO (дословно)

1. Фон белый — как был в оригинальном Variant D
2. Карточки белые, фото на белом/светлом фоне
3. Окантовки фиолетовые при hover — нравятся, оставить
4. Градиент (где используется) — как на скриншоте из брендбука: 80% чёрного + subtle фиолетовое свечение
5. Фиолетовая айдентика — в нюансах: цены, окантовки, кнопки, бейджи
6. НЕ выдумывать ничего нового — всё есть в брендбуке
7. Не интерпретировать — уточнять

## Скоуп

**Включено:**
- Откат CSS custom properties на светлую палитру (--vd-bg: #FFFFFF, --vd-bg-alt: #F5F5F7, --vd-text: #1D1D1F и т.д.)
- Откат всех hardcoded rgba-значений на светлые (borders, shadows, header, mobile menu, forms, scrollbar)
- Возврат footer на тёмный фон (контраст со светлой страницей)
- Hero-карточка: может использовать тёмный градиент из брендбука (чёрно-фиолетовый) с белым текстом
- Акценты остаются: gradient-text для цен, hover-окантовки на карточках, кнопки с градиентом
- Удаление hero-glow (ambient glow не нужен на светлом фоне)

**НЕ включено:**
- Изменение HTML-контента (товары, контакты, структура — всё остаётся)
- Изменение i18n файлов (уже исправлены)
- Изменение изображений

## Рассмотренные подходы

**Вариант A: Полный откат CSS к оригиналу S009 + точечные правки**
- Плюсы: гарантированно рабочий результат, минимум риска
- Минусы: потеряем новые стили для accessory-product карточек и product-image img

**Вариант B: Точечный откат только root variables + hardcoded rgba**
- Плюсы: сохраняем новые стили (accessory-product, product-image), меньше изменений
- Минусы: можно пропустить что-то

**Выбрано:** Вариант A. Берём оригинальный CSS из git (до тёмной переделки), добавляем только новые блоки (accessory-product, product-image img, hero-product-img). Причина: гарантия что ничего тёмного не останется.

## Технические требования

1. Восстановить :root переменные из оригинала S009
2. Восстановить все hardcoded rgba из оригинала
3. Сохранить новые CSS-блоки: `.bento-card--accessory-product`, `.product-image img`, `.hero-product-img`
4. `.bento-accessories` grid: `repeat(4, 1fr)` (не 3)
5. `.hero-glow` — удалить (не нужен на светлом)
6. Hero-карточка `.bento-card--hero-main`: рассмотреть тёмный градиент bg с белым текстом (акцент)
7. Footer: тёмный фон (контраст)
8. Inline styles в HTML — уже убраны, НЕ возвращать
